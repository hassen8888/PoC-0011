# PoC-0011_Request（方向を見ないEntry & 継続動意ExitのPoC要求仕様）

## 1. PoCの目的

**「動意判定 EA（Move/NoMove Classifier）」の検証**

方向を一切見ない Entry 条件でポジションを持ち、first-touch（±0.03）以降の「継続動意」を用いて Exit（利確・手仕舞い）条件の特徴・ルール候補を見出す。

### 1.1 PoC-0010 からの転換点

| 項目 | PoC-0010 | PoC-0011 |
|-----|---------|---------|
| **対象** | 方向予測（UP/DOWN） | 動意判定（Move/NoMove） |
| **Entry** | スコア方向に従う | 動意の強さのみ（方向無視） |
| **Exit** | なし（first-touch時点） | first-touch後の継続動意で調整 |
| **成果** | 方向予測は無意味と判定 | 継続動意ベースExit ルール候補 |

### 1.2 成果物

- Entry条件（動意ベース）の妥当性評価
- first-touch後の伸び幅の統計
- 継続動意に基づくExitルール候補（複数パターン）
- 方向を見ないEntry＋継続動意Exitの期待値・勝率・RRの概観

## 3. Entryロジック（方向を見ないEntry条件）

### 3.1 基本方針

- **Entry時点では方向を一切見ない**
  - UP/DOWN の予測は統計的に無意味（PoC-0010 で証明済み：的中率 ≈ 48%）
- **Entry条件は「動く準備が整っている状態（動意が高い状態）」のみ**
  - 複数の指標が同時に「動きの兆し」を示す状態
- **Buy/Sell方向は first-touch で事後的に決まる（分析用ラベル）**
  - Entry時点でニュートラル
  - first-touchで初めて方向が確定

### 3.2 Entry判定に使う情報

#### **A. 10秒足の動意指標（中心）**

**値幅フィルタ**
- 直近10秒足の値幅（High-Low） ≥ **0.0020**（20pips）
  - 根拠：PoC-0010で観測された動きの最小単位
  - 目的：明らかなノイズレベルを除外

**MACD強度フィルタ**
- 10秒足MACD（Histogram）の絶対値 ≥ **0.00005**
  - パラメータ：EMA(12, 26, 9)
  - 根拠：動意の方向性判定の中核指標
  - 目的：モメンタムの転換を検知

**STOCH 過度フィルタ**
- 10秒足STOCH（%K）が極端値：**%K ≤ 20 または %K ≥ 80**
  - パラメータ：(14, 3, 3)
  - 根拠：過度な状態からの反転 = 動きの準備完了
  - 目的：反転圧力の検知

**10秒ボラティリティ フィルタ**
- 直近10秒の標準偏差 ≥ **0.00008**（0.8pips）
  - 目的：ボラティリティの急上昇を検知

#### **B. スプレッド条件（必須）**

- **Spread ≤ 3**
  - 根拠：PoC-0010で実証
    - Spread ≤ 3：動きやすい（到達時間 463秒）
    - Spread > 3：レンジ傾向（到達時間 385〜437秒）
  - 目的：流動性が高い時点でのみエントリー

#### **C. 時間帯条件**

**除外時間帯：** 22:00 ～ 23:59（UTC）
- 根拠：PoC-0010 10秒足統計
  - 22時：レンジ率（0）= **23.4%**（他の時間帯の6倍以上）
  - 23時：レンジ率（0）= **18.9%**
  - 対比：2〜20時 ≈ 1〜5%
- 目的：流動性が枯渇する時間帯を回避

**優先時間帯：** 2:00 ～ 21:00（UTC）
- 全時間帯を対象にするが、22〜23時は除外
- 特に欧州〜NY の重なる時間帯（9:00 ～ 17:00）

#### **D. 直近レンジ判定（補助）**

**60秒レンジ除外**
- 直近60秒の値幅（High-Low） ≥ **0.0010**（10pips）
  - 目的：小さな往復のみ状態（完全なレンジ）を除外
  - 注：値幅が小さい = 値動きが停止している

### 3.3 Entryトリガー（具体値）

**以下の全ての条件を同時に満たしたTickで Entry 発生をログ記録：**

```
条件1：10秒値幅フィルタ
        直近10秒足（High-Low） ≥ 0.0020 ✓

条件2：MACD強度フィルタ
        abs(MACD_Histogram[10s]) ≥ 0.00005 ✓

条件3：STOCH過度フィルタ
        %K[10s] ≤ 20 または %K[10s] ≥ 80 ✓

条件4：ボラティリティフィルタ
        StdDev[10s] ≥ 0.00008 ✓

条件5：スプレッド条件
        Spread ≤ 3 ✓

条件6：時間帯フィルタ
        NOT (22:00 ≤ Hour ≤ 23:59) ✓

条件7：60秒レンジ除外
        Range[60s] ≥ 0.0010 ✓

→ 全て満たす ⇒ Entry 発生、Entry ID を記録
```

### 3.4 Entry 時点で記録する情報

- Entry ID（一意）
- Entry時刻（yyyymmdd hh:mm:ss.sss）
- Entry価格（Bid/Ask/Mid）
- 10秒足の状態（値幅、MACD、%K、%D、StdDev）
- 1分足・5分足の簡易状態（傾き、MACD、%K）
- スプレッド
- 時間帯ラベル

## 4. first-touch 定義と方向ラベル

### 4.1 first-touch の定義

**Entry価格を基準に：**

- **+0.03 到達** → 上方向 first-touch（direction = UP）
  - Entry価格 + 0.03 以上に到達した時点
  
- **-0.03 到達** → 下方向 first-touch（direction = DOWN）
  - Entry価格 - 0.03 以下に到達した時点
  
- **規定時間（600秒 = 10分）以内にどちらにも触れない** → NoTouch（direction = NONE）
  - 根拠：PoC-0010の p75 ≈ 460秒、安全マージン設定
  - 判定：600秒経過時点で両方向未到達 = 「動かない」と判定

### 4.2 方向ラベル

| first-touch | 方向ラベル | 解釈 |
|----------|----------|------|
| +0.03到達 | **UP** | 上方向の動き確定 |
| -0.03到達 | **DOWN** | 下方向の動き確定 |
| 600秒未到達 | **NONE** | レンジ状態（動かない） |

### 4.3 first-touch 時点で記録する情報

- Entry ID
- first-touch方向（UP / DOWN / NONE）
- 到達時間（秒）
- 到達までの最大逆行幅（反対方向への値動き）
- first-touch時点の10秒足/1分足/5分足の状態
  - MACD、%K、%D、値幅、傾き

## 5. 継続動意（Exit候補）の定義

### 5.1 継続動意の目的

**first-touch で方向が確定した後、「その方向にどれだけ伸びるか」を評価する**

- first-touch時点で +0.03 に到達したことが確定
- その後、さらに伸び続けるのか？
- どの程度の利確幅を設定すべきか？
- 上位足の環境認識は伸びを支持しているか？

→ これらを定量的に記録し、**Exit ルール候補を検証する**

### 5.2 継続動意に使う時間足と役割

#### **10秒足**
- **役割：** 短期動意（first-touch直後の値動き）
- **観察対象：**
  - first-touch後の直近値幅
  - MACD/STOCH の推移（加速 or 減速）
  - 最初の20秒間の方向性

#### **1分足**
- **役割：** 中期動意（伸びやすさのバイアス）
- **観察対象：**
  - first-touch時点でのトレンド方向
  - MACDの方向整合性（first-touch方向と一致/逆）
  - 傾き（上昇トレンド中か、下降トレンド中か）

#### **5分足**
- **役割：** 背景トレンド（伸びの上限・持続性）
- **観察対象：**
  - 大きな背景トレンド
  - 抵抗線・サポート線の存在
  - 中期的な環境認識

### 5.3 継続動意の指標候補

#### **1分後観測ウィンドウ（推奨）**

first-touch後 60秒間の値動きを観測：

**同方向伸び幅**
```
= max(price[first_touch → +60秒]) - price[first_touch]

例） first-touch = UP（+0.03達成）
     その後60秒間で +0.05 まで上昇
     → 同方向伸び幅 = +0.02
```

**逆方向最大逆行幅**
```
= abs(min(price[first_touch → +60秒]) - price[first_touch])

例） first-touch = UP（+0.03達成）
     その後最大 -0.01 まで下がった
     → 逆方向逆行幅 = 0.01
```

**伸び率（重要指標）**
```
= 同方向伸び幅 / max(逆方向逆行幅, 0.001)

例） 伸び幅 0.02 / 逆行幅 0.01 = 2.0
     → 伸びやすい状態

     伸び幅 0.005 / 逆行幅 0.015 = 0.33
     → 逆行が強い、伸び弱い
```

**累積値幅（時系列）**
- 同方向への累積値幅（1分間で何pips進んだか）
- 逆方向への累積値幅（往復の激しさ）

#### **10秒足MACD/STOCHの変化**

**dMACD（MACD Histogram の変化）**
```
= MACD_Hist[first_touch] - MACD_Hist[+60秒]

正の値 = モメンタムが減速（動意が弱まる）
負の値 = モメンタムが加速（動意が強まる）
```

**d%K（STOCH%K の変化）**
```
= %K[first_touch] - %K[+60秒]

利用方法：方向転換の兆候を検知
```

#### **1分足・5分足の方向整合性**

**1分足の傾き**
- first-touch時点での直近の傾き（上昇 / 下降 / 水平）
- first-touch方向との一致度：
  - 一致（alignment = 1.0）
  - 逆行（alignment = -1.0）
  - 中立（alignment = 0）

**5分足の傾き**
- 背景トレンドの方向
- first-touch方向との一致度

### 5.4 継続動意ログの構造

each Entry に対して、以下を記録：

```
[Entry ID: #00001]
├─ first-touch情報
│   ├─ 方向：UP
│   ├─ 到達時刻：hh:mm:ss
│   └─ 到達時の10s/1m/5m状態
│
└─ 継続動意観測（+60秒時点）
    ├─ 同方向伸び幅：0.0025
    ├─ 逆方向逆行幅：0.0008
    ├─ 伸び率：3.1
    ├─ 10秒MACD変化：-0.00003
    ├─ 10秒STOCH変化：-15
    ├─ 1分足傾き：UP（alignment = 1.0）
    ├─ 5分足傾き：UP（alignment = 1.0）
    └─ 総合判定：継続動意 HIGH
```

## 6. Exit ロジック（継続動意ベース）

### 6.1 Exit判定の基本戦略

**継続動意ベースの段階的Exit**

first-touch 後 60秒時点での「伸び率」により、以下のルール候補を検証：

#### **パターンA：伸び率ベースExit（推奨）**

```
first-touch = UP の場合：

伸び率 ≥ 2.5 → 高い継続動意
         → Entry + 0.05 で利確（多くの場合そこまで伸びる）
         根拠：逆行が小さく、伸びが大きい状態

1.5 ≤ 伸び率 < 2.5 → 中程度の継続動意
         → Entry + 0.04 で利確
         根拠：バランスの取れた伸び

1.0 ≤ 伸び率 < 1.5 → 弱い継続動意
         → Entry + 0.03 で即利確（first-touch時点で利確）
         根拠：逆行が強く、伸びが期待できない

伸び率 < 1.0 → 逆行強い（損失リスク大）
         → Entry + 0.03 で即利確 または 損切り検討
         根拠：反転の兆候、延長しない
```

#### **パターンB：1分足・5分足整合性ベースExit（補助）**

```
継続動意の追加判定：

1分足と5分足が両方ともfirst-touch方向と一致
 → より高い確度で伸びやすい（さらに利確幅拡大可能）

1分足は一致、5分足は逆行
 → 短期トレンド継続も、背景トレンドに反発するリスク
   （利確幅を控えめに）

1分足と5分足の両方が逆行
 → 強い反発圧力（即座に利確推奨）
```

#### **パターンC：時間ベースExit（フォールバック）**

```
+3分（180秒）で未決着
 → 強制EXIT（Entry + 0.03 または損切り）

根拠：
- PoC-0010で p75 ≈ 460秒
- 最初の1分で伸びが限定的 = エネルギー不足の兆候
```

### 6.2 Exit判定の実装順序

```
Step 1：first-touch方向の確定（+0.03到達）
        ↓
Step 2：60秒後のスナップショット取得（伸び率計算）
        ↓
Step 3：伸び率レベルで基本的なExit幅を決定
        ↓
Step 4：1分足・5分足の整合性で調整
        ↓
Step 5：実際の約定記録
```

## 付録A: PoC-0010 からの根拠データサマリー

### A.1 時間帯別レンジ率（0 の発生率）

```
10秒足統計（全29,001サンプル）より：

22:00-22:59 → 0率 = 23.4% ★除外推奨
23:00-23:59 → 0率 = 18.9% ★除外推奨

2:00-20:59  → 0率 < 3%   ★良好
21:00-21:59 → 0率 = 17.7% ? 警戒

→ 決定：22時～23時59分を除外
```

### A.2 スプレッド別特性

```
10秒足統計より：

Spread ≤ 3.0：
  - サンプル：3,574
  - 到達時間：463秒（最適）
  - 0率：4.4%（低い）
  → 流動性は低いが、動く確率は高い

Spread 3.0-5.0：
  - サンプル：12,350
  - 到達時間：396秒
  - 0率：3.1%
  → バランス

Spread 5.0+：
  - サンプル：5,854
  - 到達時間：437秒
  - 0率：6.1%
  → 流動性が落ちると0が増える

→ 決定：Spread ≤ 3 で Entry
```

### A.3 first-touch 時間分布

```
10秒足統計（up / down 両方向）：

mean：346-375秒
median：239-248秒
p75：462-496秒

→ 600秒（10分）は p75 + 104秒 のマージン
→ 安全な NoTouch 判定時間
```

### A.4 方向予測の失敗

```
10秒足での direction_relation より：

score_dir = -1（DOWN予測）の場合：
  - 的中率：4,195/8,875 = 47.3%

score_dir = 1（UP予測）の場合：
  - 的中率：9,412/19,126 = 49.2%

平均：≈48% ← ランダム（50%）以下

結論：方向予測は統計的に意味がない
     → PoC-0011では方向を見ない戦略に転換
```
